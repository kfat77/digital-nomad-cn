## Security Policy

### Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

### Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**DO NOT** open a public issue.

Instead, send an email to: **security@globalmobility.io**

We will respond within 48 hours and work with you to assess and address the issue.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Security best practices for contributors

- Never commit API keys, passwords, or tokens
- Use environment variables for sensitive data
- Keep dependencies updated
- Validate all user inputs
- Sanitize data before rendering

### Acknowledgments

We will publicly acknowledge responsible disclosures (with your permission) in our security advisory.
