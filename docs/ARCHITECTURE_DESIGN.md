# Global Mobility Infrastructure — 网站架构设计 v2.0
## GitHub 爆款项目级设计

---

## 设计哲学

不是好看，而是**有用**。不是营销网站，而是**基础设施**。每个页面必须回答一个问题：用户来了，能得到什么？

**爆款原则**：
- 数据为王：每个页面都有实时/结构化数据
- 开源气质：GitHub 感、开发者友好、可 Fork、可贡献
- 性能优先：首屏 < 1s，Lighthouse 90+
- 移动端优先：60% 用户来自移动端
- SEO 原生：每个页面都是可索引的独立内容

## 技术架构

### 前端
- **框架**: 纯静态 HTML + Vanilla JS (无框架，极致性能)
- **样式**: Tailwind CSS (按需构建)
- **3D**: Three.js (仅首页地球)
- **图表**: Chart.js (数据可视化)
- **地图**: Leaflet.js (城市/国家地图)
- **搜索**: Lunr.js (客户端搜索)
- **构建**: Vite (构建工具)

### 数据
- **存储**: JSON 文件 (静态)
- **构建时**: Node.js 脚本生成静态页面
- **更新**: GitHub Actions 自动化
- **CDN**: Cloudflare / GitHub Pages

### 部署
- **静态托管**: GitHub Pages (docs/ 目录)
- **CDN**: Cloudflare (自定义域名 + 缓存)
- **CI/CD**: GitHub Actions (推送自动构建)
- **分析**: Plausible (隐私友好分析)

### SEO 技术
- **SSG**: 构建时生成所有页面
- **Sitemap**: 自动生成 XML sitemap
- **RSS**: 文章 RSS 订阅
- **Structured Data**: 每个页面注入 JSON-LD
- **Open Graph**: 动态生成 OG 图片
- **Canonical**: 每页唯一 canonical
- **Hreflang**: 多语言支持标记
- **Robots**: 动态生成 robots.txt

## 设计原则检查清单

每个页面设计时检查：

- [ ] 用户来了，能得到什么？（价值主张清晰）
- [ ] 数据是否完整？（无占位符，有真实数据）
- [ ] 首屏是否有核心信息？（首屏 < 1s）
- [ ] 移动端是否可用？（响应式优先）
- [ ] 是否有明确的 CTA？（至少一个行动号召）
- [ ] SEO 是否完整？（Title/Description/Structured Data）
- [ ] 是否可访问？（键盘导航、屏幕阅读器、高对比度）
- [ ] 是否可分享？（Open Graph/Twitter Card）
- [ ] 是否有数据纠错入口？（社区贡献）
- [ ] 是否链接到相关页面？（内链建设）
