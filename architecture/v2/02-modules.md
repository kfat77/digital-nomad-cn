# 第二部分：项目模块拆分

## 目标：从单一网站 → 多模块基础设施

```
┌─────────────────────────────────────────────────────────────┐
│              Digital Nomad CN — Module Architecture         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                  CORE LAYER (核心层)                   │  │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐     │  │
│  │  │ schemas │ │datasets │ │  api    │ │   ai    │     │  │
│  │  │ (数据定义)│ │ (数据集) │ │ (API服务)│ │(AI能力) │     │  │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘     │  │
│  │                                                        │  │
│  │  核心原则：即使没有网站，这四层也可以独立存在           │  │

│                                                             │
│  ┌───────────────────────────────────────────────────────┐  │

│                                                             │
```

## 模块定义

### 1. Website（网站）

| 属性 | 说明 |
|------|------|
| **存在意义** | 面向普通用户的内容消费界面 |
| **核心职责** | 展示数据、提供搜索、生成SEO页面 |
| **数据来源** | 从 `schemas/` + `datasets/` 自动生成 |
| **技术栈** | 静态生成器 (11ty/Astro) + GitHub Pages |
| **可替代性** | 高——任何前端都可以消费API重建 |
| **独立性** | ❌ 依赖数据层 |

```
┌─────────────────────────────────────┐
│  Website                            │
│  ┌───────────────────────────────┐  │
│  │  页面 = f(数据集)              │  │
│  │  不存储数据，只消费数据         │  │
│  │  构建时从 schemas/ 读取        │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

---

### 2. Database（数据库）

| 属性 | 说明 |
|------|------|
| **存在意义** | 项目最核心的资产——结构化Global Mobility数据 |
| **核心职责** | 存储、验证、版本控制所有国家/城市/签证/税务等数据 |
| **存储形式** | JSON文件（Git版本控制）|
| **技术栈** | JSON + JSON Schema + Git |
| **可替代性** | 零——这是项目存在的根本 |
| **独立性** | ✅ 完全独立 |

```
┌─────────────────────────────────────┐
│  Database (schemas/ + datasets/)    │
│  ┌───────────────────────────────┐  │
│  │  country.json — 177个国家      │  │
│  │  city.json    — 300+城市       │  │
│  │  visa.json    — 签证体系       │  │
│  │  tax.json     — 税务规则       │  │
│  │  ... 15+ 数据文件              │  │
│  └───────────────────────────────┘  │
│  用Git做版本控制，用Schema做校验     │
└─────────────────────────────────────┘
```

### 3. API（应用程序接口）

| 属性 | 说明 |
|------|------|
| **存在意义** | 让数据可被程序化访问 |
| **核心职责** | 提供标准化的数据查询接口 |
| **接口形式** | REST / GraphQL / MCP / Webhook |
| **技术栈** | Cloudflare Workers / Node.js |
| **可替代性** | 中——协议标准化后可替换实现 |
| **独立性** | ✅ 可独立部署 |

```
┌─────────────────────────────────────┐
│  API Layer                          │
│  ┌───────────────────────────────┐  │
│  │  REST   → GET /api/v1/countries│  │
│  │  GraphQL → query { countries } │  │
│  │  MCP    → tools/list          │  │
│  │  Webhook → 数据变更事件推送    │  │
│  └───────────────────────────────┘  │
│  所有接口都从 schemas/ 读取数据      │
└─────────────────────────────────────┘
```

## 模块依赖关系

```
                    ┌─────────────┐
                    │   schemas   │ ←── 唯一真相源
                    │  (数据定义)  │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ▼               ▼               ▼
    ┌────────────┐  ┌────────────┐  ┌────────────┐
    │  datasets  │  │    api     │  │    ai      │
    │  (数据集)   │  │  (API服务)  │  │ (AI能力)   │
    └─────┬──────┘  └─────┬──────┘  └─────┬──────┘
          │               │               │
          │    ┌──────────┴──────────┐    │
          │    │                     │    │
          ▼    ▼                     ▼    ▼
    ┌─────────────────────────────────────────┐
    │         CONSUMER LAYER                  │
    │  ┌────────┐ ┌────────┐ ┌────────┐      │
    │  │website │ │  SDK   │ │  CLI   │      │
    │  └────────┘ └────────┘ └────────┘      │
    │  ┌────────┐ ┌────────┐ ┌────────┐      │
    │  │  MCP   │ │examples│ │  Docs  │      │
    │  └────────┘ └────────┘ └────────┘      │
    └─────────────────────────────────────────┘
```

**关键原则：**
- `schemas/` 是所有模块的**唯一真相源**
- 消费层模块**不互相依赖**
- 任何一个消费层都可以被替换，不影响其他模块
- 数据层变更 → 自动触发所有消费层更新
