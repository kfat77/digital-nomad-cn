### 角色定义

| 角色 | 权限 | 职责 | 数量目标 |
|------|------|------|----------|
| **核心维护者** | 合并PR、发布版本、架构决策 | 审核重大变更、维护CI/CD、处理争议 | 5-10人 |
| **国家维护者** | 审核特定国家数据PR | 维护国家数据准确性、回答相关问题 | 50-100人 |
| **城市维护者** | 审核特定城市数据PR | 维护城市数据、更新本地信息 | 100-200人 |
| **数据贡献者** | 提交数据PR | 修正错误、补充数据、更新时效信息 | 500+人 |
| **代码贡献者** | 提交代码PR | 改进工具、修复bug、新功能 | 不限 |
| **文档贡献者** | 提交文档PR | 翻译、纠错、改进说明 | 不限 |

```
- 每月至少审查1个泰国数据PR
- 每季度更新一次泰国核心数据
- 回答关于泰国的Issue
```

## 数据审核流程

```
┌─────────────────────────────────────────────────────────┐
│                    数据PR审核流程                        │
│                                                         │
│  1. 贡献者提交PR                                         │
│     └─ 自动触发 GitHub Actions                          │
│                                                         │
│  2. 自动校验                                             │
│     ├─ ✅ Schema校验（JSON是否符合Schema）               │
│     ├─ ✅ 数据完整性（必填字段是否齐全）                 │
│     ├─ ✅ 引用一致性（外键引用是否存在）                 │
│     ├─ ✅ 数值范围（数值是否在合理范围）                 │
│     ├─ ✅ 无占位符（是否有TODO/placeholder）             │
│     └─ ✅ 来源标注（数据是否有来源）                     │
│                                                         │
│  3. 生成审核报告                                         │
│     ├─ 变更摘要（改了哪些字段）                          │
│     ├─ 影响分析（影响哪些国家和城市）                    │

│     ├─ 数据来源是否可靠？                                │

│  6. 合并                                                 │
│     ├─ 自动更新索引                                      │
│     ├─ 自动生成派生数据                                  │
│     ├─ 自动更新API缓存                                   │
```

## 自动检查 (CI/CD)

```yaml
# .github/workflows/data-validation.yml
name: Data Validation

on:
  pull_request:
    paths:
      - 'datasets/**'
      - 'schemas/**'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Validate JSON Schema
        run: npm run validate:schema
      
      - name: Check data integrity
        run: npm run validate:integrity
      
      - name: Check cross-references
        run: npm run validate:references
      
      - name: Check no placeholders
        run: npm run validate:no-placeholders
      
      - name: Check source attribution
        run: npm run validate:sources
      
      - name: Generate validation report
        run: npm run validate:report
      
      - name: Comment PR with report
        uses: actions/github-script@v7
        with:
          script: |
            const report = require('./validation-report.json');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## 数据校验报告\n\n${JSON.stringify(report, null, 2)}`
            });
```

## 争议处理机制

```
数据争议处理：

1. 在Issue中提出争议
   "我认为XX国家的XX数据不准确，原因是..."

2. 自动通知相关维护者
   @country-maintainer-thailand

3. 维护者7天内回应
   - 确认错误 → 提交修正PR
   - 认为正确 → 提供来源证据
   - 无法判断 → 标记为"待验证"

4. 核心维护者介入（如果争议持续）
   - 评估双方证据
   - 做出最终判断
   - 更新数据并记录原因

5. 争议记录归档
   - 保留在争议日志中
   - 供未来参考

原则：
- 数据争议用数据解决
- 来源可靠 > 个人经验
- 官方来源 > 媒体来源
- 多个来源 > 单个来源
```
