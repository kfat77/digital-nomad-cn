# 第一部分：如果网站消失，项目还能存在吗？

## 答案：不能。——这就是问题所在。

---

## 当前状态分析

### 现在的 digital-nomad-cn 是什么？

```
┌─────────────────────────────────────────┐
│           GitHub Pages 网站              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │  index  │ │ country │ │  city   │   │
│  │ .html   │ │  /XX/   │ │  /XX/   │   │
│  └─────────┘ └─────────┘ └─────────┘   │
│                                         │
│  数据 = 嵌入在HTML中的JavaScript对象      │
│  页面 = 手工编写或脚本生成的静态HTML      │
│  部署 = GitHub Pages (docs/目录)         │
└─────────────────────────────────────────┘
```

**关键问题：**
- `country-data.js` 中的数据是 **内联在HTML中的JavaScript对象**
- 如果删除 `website/` 目录，**数据就消失了**
- 没有独立的Data Repository
- 没有API层
- 没有数据版本控制
- 没有标准化Schema

### 这意味着什么？

| 场景 | 结果 |
|------|------|
| GitHub Pages 宕机 | 整个项目不可访问 |
| 想做一个App | 没有API可用，必须从HTML里爬数据 |
| 想做一个数据分析 | 没有结构化数据，只能解析HTML |
| 想接入ChatGPT Plugin | 没有标准化的数据源 |
| 别人想Fork数据 | 拿到的是一堆HTML，不是数据 |
| 5年后回看 | 页面可能还在，但数据格式已经过时 |

## 关键区别

### V1：网站 = 项目

```
项目存在 ⟺ 网站在线
数据 ⟹ 嵌入在HTML中
贡献 ⟹ 编辑HTML文件
消费 ⟹ 访问网页
```

### V2：基础设施 = 项目

```
项目存在 ⟺ 数据集 + API + 社区 健康
数据 ⟹ 独立的结构化JSON文件（版本控制）
贡献 ⟹ 提交数据PR（通过Schema校验）
消费 ⟹ API调用 / SDK调用 / AI Agent调用 / 网站浏览
网站 ⟹ 只是众多消费者之一
```

## 验证标准

### "Infrastructure" 的检验标准

| 检验项 | V1 | V2目标 |
|--------|-----|--------|
| 关闭网站后，数据是否仍然可用？ | ❌ 不可 | ✅ 可以（数据在schemas/） |
| 第三方开发者能否调用数据？ | ❌ 不能 | ✅ 可以（REST/GraphQL/MCP） |
| AI Agent能否直接读取数据？ | ❌ 不能 | ✅ 可以（MCP Server + Structured Output） |
| 数据是否有版本历史？ | ❌ 没有 | ✅ 有（Git + Schema版本） |
| 数据变更是否有自动校验？ | ❌ 没有 | ✅ 有（CI/CD数据验证） |
| 社区能否独立贡献某国数据？ | ⚠️ 困难 | ✅ 容易（提交对应JSON文件） |
| 项目能否被其他项目依赖？ | ❌ 不能 | ✅ 可以（npm/pip包） |

---

## 结论

> **当前的 digital-nomad-cn 仍然只是一个网站，不是基础设施。**
>
> V2 的目标是让数据、API、社区成为项目的核心，
> 网站只是这些核心资产的**一个消费者界面**。
>
> 最终状态：
> - `schemas/` 目录可以独立存在
> - `api/` 可以独立部署
> - `website/` 可以随时重写，不影响数据
> - AI Agent可以直接调用MCP Server获取数据
> - 全球开发者可以依赖我们的npm/pip包

## 迁移路径

```
Phase 1 (M1-M3):  数据层抽离
  └─ 把嵌入HTML的数据提取为独立JSON
  └─ 定义完整的Schema
  └─ 建立数据→页面的自动生成管道

Phase 2 (M4-M6):  API层建设
  └─ 部署REST API
  └─ 部署GraphQL API
  └─ 发布MCP Server

Phase 3 (M7-M9):  生态建设
  └─ SDK发布 (npm/pip)
  └─ 贡献者流程自动化
  └─ AI Agent集成

Phase 4 (M10-M12): 规模化
  └─ 多语言数据覆盖
  └─ 第三方依赖增长
  └─ 社区自治
```
